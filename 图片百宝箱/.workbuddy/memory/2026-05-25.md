# 2026-05-25

## 图片百宝箱 - 修复 ads.txt 未找到

- **问题**：Google AdSense 后台显示 ads.txt 状态"未找到"
- **根因**：`vercel.json` 中 `"cleanUrls": true` 导致 Vercel 路由器拦截 `/ads.txt` 请求（cleanUrls 会干扰非 HTML 文件的路由处理）
- **修复**：在 `vercel.json` 中添加：
  1. `rewrites`：`/ads.txt` → `/ads.txt`，强制 Vercel 直接提供文件
  2. `headers`：为 `/ads.txt` 明确设置 `Content-Type: text/plain` 和 `Cache-Control: public, max-age=86400`
- **验证**：`curl https://img.b1y.ren/ads.txt` 返回 HTTP 200，内容正确
- **已推送到GitHub**：commit 1997c12

## 图片百宝箱 - 横向广告位空白问题排查

- **现象**：用户反馈横向广告位（ad-after-tabs）没有显示广告，呈现空白
- **根因**：`ads.txt` 中 Publisher ID 格式错误，写成了 `pub-4519467063111183`（缺少 `ca-` 前缀），正确格式应为 `ca-pub-4519467063111183`。AdSense 爬虫验证 ads.txt 时发现 ID 不匹配，拒绝投放广告。
- **修复**：将 `ads.txt` 内容修正为 `google.com, ca-pub-4519467063111183, DIRECT, f08c47fec0942fa0`
- **已推送**：commit b28d59d
- **注意**：Google AdSense 重新抓取 ads.txt 需要几小时到1天，广告位才会开始显示广告
